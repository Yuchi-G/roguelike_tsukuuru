# ローグライクツクール 要件定義

## プロダクトビジョン

**「ローグライクツクール」** — コードを書かずにローグライクゲームを作れるデスクトップアプリ

ユーザーはGUIだけで敵・アイテム・マップ・ロジックをすべて定義し、そのままゲームとしてプレイできる。

### ターゲットユーザー
- コードを書かなくてもゲームが作れるツール（他の人も使える）
- 「何でも設定できるんだな」と思ってもらえることがゴール

---

## 機能一覧

### 1. プロジェクトハブ
- 新規作成 / 開く / 最近使ったプロジェクト
- プロジェクト = ゲーム1本分のすべてのデータ

---

### 2. データベースエディタ

**敵**
- 名前・見た目・HP・MP・ATK・DEF・SPD
- スキルリスト（使用可能なスキルを紐づけ）
- AIスクリプト（ビジュアルロジックで定義）
- ドロップテーブル

**アイテム**
- 名前・見た目・種別（消耗品/装備/鍵など）
- 効果スクリプト（ビジュアルロジックで定義）
- 装備ならスロット（武器/防具/アクセサリ）とステータス補正
- ショップ価格

**スキル**
- 名前・消費MP・対象（自分/単体/全体/範囲）・クールダウン
- 効果スクリプト（ビジュアルロジックで定義）
- アニメーション（記号 or 色でCanvasに表現）

**状態異常**
- 名前・持続ターン・アイコン記号
- ターン経過効果（毎ターンHP-5など）
- ステータス修正（ATK×0.5など）
- 付与/解除条件

**タイル**
- 既存のタイル定義を継続・拡張

**プレイヤー定義**
- 初期ステータス（HP・MP・ATK・DEF・SPD）
- 初期スキル・装備スロット
- レベルアップ時のステータス成長式（ステータスごとに設定）

---

### 3. ビジュアルスクリプトエディタ

AIも・アイテム効果も・イベントも・スキルも、同じエディタで組み立てる。

**変数の種類**
- グローバル変数: ゲーム全体で共有（クエストフラグなど）
- エンティティ変数: 敵・NPCごとに持つ（行動カウンターなど）
- ローカル変数: スクリプト内だけで使う一時変数

**条件式の例**
```
HP% < 30
プレイヤーが視界内
turn_count % 3 == 0
フラグ「boss_defeated」== true
バッグにアイテム「鍵」がある
```

**アクションの例**
```
移動: プレイヤーに近づく / 逃げる / ランダム / 指定座標へ
攻撃: 通常攻撃 / スキル使用
ステータス: 変数をセット / フラグをセット
エンティティ: 状態異常を付与 / HP回復 / ダメージ
マップ: マップ遷移 / エンティティを出現させる
UI: メッセージ表示 / アイテムを渡す
```

**制御フロー**
```
IF 条件 THEN
  アクション列
ELSE
  アクション列
END

LOOP N回
  アクション列
END

WHILE 条件
  アクション列
  BREAK IF 条件
END

WAIT Nターン
```

---

### 4. マップエディタ

**マップ構造**
- マップ = 名前付きのグリッド（「洞窟1F」「ボス部屋」など）
- 生成方法: ランダム生成 or 手動配置（マップごとに選択）

**手動配置**
- タイルパレットで塗る
- 敵・アイテム・NPCを置く（それぞれスクリプトを持てる）
- 出口を置いて接続先マップを指定（分岐も可）

**マップイベント**
- タイルイベント: 特定の座標を踏んだとき発火
- オブジェクトイベント: 「調べる」で発火（NPCや宝箱など）
- 自動実行: マップ進入時に自動で発火

**マップ接続**
- 出口→入口のリンクで複数マップをつなぐ
- フラグ条件付き接続（隠し部屋など）
- 分岐ルートに対応

---

### 5. ゲームシステム設定

- 戦闘順: シンプルターン制 / SPD依存の行動順
- レベルアップ式・ステータス成長式
- バッグ・装備スロット構成
- セーブ/ロード（マップイベントで「セーブポイント」を置く）
- ゲームオーバー条件・クリア条件（スクリプトで定義）

---

### 6. プレイテスト

- エディタから直接ゲームを起動
- 特定マップ・特定フロアから開始できる

---

## プロジェクトのデータ構造

```
プロジェクト
├── データベース
│   ├── 敵定義
│   ├── アイテム定義
│   ├── スキル定義
│   ├── 状態異常定義
│   └── タイル定義
├── マップ
│   ├── マップA（手動）
│   ├── マップB（ランダム生成ルール）
│   └── マップC（手動）
│       └── 接続: マップA出口 → マップB入口
├── スクリプト
│   └── 各AIや効果やイベントが参照
└── ゲーム設定
    ├── プレイヤー定義
    └── システム設定
```

---

## 実装フェーズ（案）

### フェーズ1: スクリプトエンジン
全ての土台。変数・条件式・アクション列・制御フローのデータ構造とUIを設計する。
これが完成すれば AI・アイテム効果・スキル・イベントが同じ基盤で動く。

### フェーズ2: 戦闘の深み
MP・スキル・状態異常を追加。AIをスクリプトエンジンで再定義。

### フェーズ3: マップシステム
複数マップ・接続定義・グリッドエディタUI・ランダム生成との共存。

### フェーズ4: イベント・シナリオ
マップトリガー・NPC・会話・カスタム勝利/ゲームオーバー条件。
