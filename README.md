# Roguelike Tsukuuru

TypeScript + Vite + Electron + HTML Canvasで作る、ローグライク制作に特化したデスクトップアプリ用ミニゲームエンジンです。汎用エンジンではなく、グリッド移動、ターン制、戦闘、ダンジョン生成など、ローグライクに必要な最小機能をシンプルにまとめています。

## 現在できること

- ランダムな部屋と通路で構成されたダンジョンを生成
- `@`のプレイヤーを矢印キーまたはWASDで移動
- 壁`#`には移動不可
- 敵にぶつかると攻撃
- プレイヤー行動後に敵が1回動く完全ターン制
- HP、攻撃力、死亡、ゲームオーバーを管理
- 回復薬`!`を拾ってバッグに保管
- バッグ内の回復薬を使ってHP回復
- バッグの上限は10個。満杯時は拾うアイテムと捨てるアイテムを選択
- 武器`)`を拾うと自動装備して攻撃力を強化
- 次の階へ進む時にHP、レベル、経験値、攻撃力、武器を持ち越し
- 敵を倒すと経験値を獲得し、一定値でレベルアップ
- レベルアップ時に最大HPが増え、HPが全回復し、攻撃力が上昇
- 複数種類の敵がランダムに出現（スライム・ゴブリン・オーク・コウモリ）
- コウモリはHP50%以下になるとプレイヤーから逃げる逃走AI
- 階段`>`から次の階へ移動
- エリクサー`~`を拾ってバッグに保管し、使うとHPが全回復
- 階層ルールに応じて敵の出現重み、敵数、アイテム出現率、敵の強化量を変更
- プレイヤー周囲8マスの簡易視界
- 画面右側にLV、EXP、HP、攻撃力、武器、バッグ、階層、ログを表示
- 設定パネルで敵・アイテム・タイルの追加削除、AI/効果/階層ルール/描画/ログ文言を編集
- プロジェクトJSONをファイルとして保存/読込
- ゲーム開始前に設定を編集し、「ゲームスタート」で初めてダンジョンを生成

## セットアップ

```bash
npm install
```

## 開発起動

```bash
npm run dev
npm run desktop:dev
```

別ターミナルでViteとElectronを起動します。ブラウザ単体での利用はサポート対象外です。

## デスクトップ起動

```bash
npm run desktop
```

## ビルド

```bash
npm run build
```

## 操作

| 操作 | 内容 |
| --- | --- |
| ゲームスタート | 設定パネルの内容でゲームを開始 |
| 設定を反映して最初から | プレイ中の設定変更を反映して1階から再開始 |
| 矢印キー / WASD | 移動 |
| 敵に向かって移動 | 攻撃 |
| 回復薬の上に移動 | 回復薬をバッグに入れる |
| 武器の上に移動 | 武器を拾って自動装備 |
| H / UIの「使う」ボタン | バッグ内の回復薬を使う |
| UIの「中身を見る」ボタン | バッグの中身を表示 |
| 階段の上でSpace | 次の階へ進む |
| ゲームオーバー時にEnter | 新しいゲームを開始 |

## 成長システム

プレイヤーは敵を倒すと経験値を獲得します。初期状態はLv.1、EXP 0、次のレベルまで10 EXPです。

レベルアップ時は以下のように成長します。

| 項目 | 変化 |
| --- | --- |
| LV | +1 |
| EXP | 必要経験値分を消費 |
| 次の必要EXP | 現在値の1.5倍、切り捨て |
| 最大HP | +5 |
| 現在HP | 最大HPまで回復 |
| 攻撃力 | +2 |

## 敵の種類

敵は階層ルールの出現重みに応じて選ばれます。階層ごとのHP/攻撃力補正も設定できます。

| 種類 | 記号 | HP | 攻撃力 | EXP | AI |
| --- | --- | ---: | ---: | ---: | --- |
| スライム | `s` | 8 | 2 | 4 | chase |
| ゴブリン | `g` | 10 | 3 | 5 | chase |
| オーク | `O` | 16 | 5 | 9 | chase |
| コウモリ | `b` | 6 | 2 | 3 | flee |

コウモリはHPが最大値の50%以下になるとプレイヤーから逃げます。通常時はchaseと同じ動作をします。

## 装備システム

プレイヤーは武器スロットを1つ持ちます。武器を拾うと自動で装備され、基礎攻撃力に武器の攻撃力が加算されます。すでに武器を装備している場合、古い武器はバッグへ返却されます（バッグ満杯時はロスト）。

| 武器 | 記号 | 攻撃力 |
| --- | --- | ---: |
| 剣 | `)` | +3 |

## アイテムバッグ

プレイヤーはアイテムをバッグに入れて持ち歩けます。上限は10個です。`H`キーまたはUIの「使う」ボタンで、バッグ内のアイテムを1つ使います。UIの「中身を見る」ボタンでバッグ内のアイテム一覧を表示できます。

| アイテム | 記号 | 効果 |
| --- | --- | --- |
| 回復薬 | `!` | HPを20回復 |
| エリクサー | `~` | HPを最大まで全回復 |

バッグが満杯の時に回復薬を拾うと、マップ画面上に半透明の選択UIが表示され、以下を選べます。

| 選択 | 内容 |
| --- | --- |
| これを捨てる | 選んだバッグ内アイテムを捨て、拾った回復薬をバッグに入れる |
| 拾わず捨てる | 拾った回復薬をバッグに入れず捨てる |

## ツクール風設定

サンプルゲーム固有の値は [src/game/sampleGameConfig.ts](src/game/sampleGameConfig.ts) にまとめています。

設定で変更できる主な項目です。

| 項目 | 内容 |
| --- | --- |
| `player` | プレイヤー名、見た目、初期HP、攻撃力、レベル、バッグ上限 |
| `dungeon` | マップサイズ、部屋数、部屋サイズ |
| `tiles` | 壁、床、階段、追加タイルの見た目、通行可否、散布率 |
| `enemies` | 敵の名前、見た目、HP、攻撃力、経験値、AI ID |
| `items` | アイテムの名前、見た目、効果ID、効果パラメータ |
| `floorRules` | 階層ごとの敵数、敵出現重み、アイテム出現率、敵強化量 |
| `render` | Canvasのタイルサイズ、フォント、背景色、探索済み色、ゲームオーバー色 |
| `fov` | 視界半径 |
| `progression` | レベルアップ時の必要EXP倍率、HP増加、攻撃力増加 |
| `messages` | ログ文言 |
| `hooks` | 攻撃、死亡、取得、階層移動、ゲームオーバー時のイベント |
| `customAiIds` | 設定UIのAIセレクトに追加するカスタムAI ID一覧 |
| `customEffectIds` | 設定UIの効果セレクトに追加するカスタム効果ID一覧 |

エンジン側はターン進行、衝突判定、描画、入力、マップ管理、FOV、ログ、基本戦闘処理を持ち、サンプルゲーム側は設定と必要な独自処理を持つ構成です。

## 設定UI

画面右側の「設定」パネルから、以下の設定を変更できます。

- プレイヤー名、見た目、初期HP、攻撃力、初期レベル、次レベル経験値、バッグ上限
- ダンジョンサイズ、部屋数、部屋サイズ
- 成長式、FOV半径
- Canvas描画スタイル
- 敵の追加削除、名前、見た目、色、HP、攻撃力、経験値、AI
- アイテムの追加削除、名前、見た目、色、効果、効果値
- 階層ごとの敵数、敵出現重み、アイテム出現率、敵強化量
- 壁、床、階段、追加タイルの見た目、文字色、背景色、通行可否、散布率
- 敵ごとに各階層ルールでの出現重みを編集
- ログ文言
- プロジェクトJSONファイルの新規/開く/保存/名前を付けて保存/初期化

「設定を反映」を押すと、現在の設定を使って1階から新しいゲームを開始します。設定の保存はプロジェクトJSONファイルとして行います。プレイ中のセーブデータ保存ではありません。

## 拡張ポイント

敵AIは `AiRegistry` に登録し、敵データの `aiId` から呼び出します。標準では `chase`、`stationary`、`random` を用意しています。`chase` の移動ロジックは `chaseMove()` としてエクスポートされており、独自AI内で再利用できます。

アイテム効果は `ItemEffectRegistry` に登録し、アイテムデータの `effectId` と `params` で呼び出します。標準では `heal` と `equipWeapon` を用意しています。

独自AIや独自効果は `main.ts` でレジストリに登録します。設定UIのセレクトに表示するには、`GameConfig` の `customAiIds` / `customEffectIds` にIDを追加してください。

```ts
// main.ts での登録例
game.aiRegistry.register("flee", ({ game: g, enemy }) => {
  if (enemy.hp <= enemy.maxHp * 0.5) {
    // 逃走ロジック
  } else {
    chaseMove(g, enemy); // 通常時はchaseを再利用
  }
});

game.itemEffectRegistry.register("fullHeal", ({ game: g, player, itemName, source }) => {
  if (source === "pickup") { g.offerBagItem({ ... }); return; }
  player.heal(player.maxHp);
});
```

```ts
// sampleGameConfig.ts でUI表示を有効化
customAiIds: ["flee"],
customEffectIds: ["fullHeal"],
```

## 設計メモ

- 外部ゲームライブラリは使わず、Canvas APIで描画しています。
- ファイル操作はElectron main processが担当し、rendererはpreload経由のAPIだけを使います。
- エンジン部分は`src/engine`（責務別サブディレクトリ）、アプリUI・保存は`src/app`、サンプルゲーム固有の実装は`src/game`に分けています。
- 標準の`chase` AIは最短経路探索を使わず、プレイヤーとのx/y差分で単純に近づきます。
- 初期MVPのため、プレイ中のセーブデータ、オンライン機能、3D描画は実装していません。
